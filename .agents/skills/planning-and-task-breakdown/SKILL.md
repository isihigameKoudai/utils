# Planning and Task Breakdown

## ゴール

- 仕様の受け入れ基準が、依存順に並んだタスク列に漏れなく対応づいている
- どのタスクも単独で完了判定できる（実行可能なコマンド or 観察可能な挙動が完了条件）
- 計画が `doc/plans/<topic>.md` に置ける Markdown として出力され、ユーザー承認後に実装が始まる

## 手順

### 1. 読むだけ。書かない

計画フェーズでは仕様（`doc/plans/<topic>.md`）と関連コードを読み、依存関係と既存パターンを把握する。コードの変更はしない。配置先の判断材料:

- `utils/` = 再利用ユーティリティ（テストは同ディレクトリの `*.test.ts`）
- `src/` = features アーキテクチャ。依存方向は routes → features → components → utils の単方向のみ

### 2. 依存グラフを描き、ブロッカーを特定する

タスク間の「先に終わっていないと始められない」関係を洗い出す。次のものはブロッカーとして **先頭に置く**:

- 他の全タスクが乗る基盤（共通ユーティリティ・型定義・設定変更）
- 失敗したら計画ごと見直しになる高リスク項目（新規依存の互換性確認、Vite 8 / Rolldown でのビルド可否など）— fail fast

### 3. タスクに切る

粒度の基準:

| 基準 | 内容 |
|---|---|
| 1 タスク = 1 コミット相当 | 終わった時点でコミットできる。中途半端な壊れた状態で終わらない |
| 単独検証可能 | そのタスクだけで `pnpm vitest run <file>` / `pnpm run build` / 操作手順による確認ができる |
| タイトルに「と」が入らない | 「A と B を実装」は 2 タスク |
| 横切りより縦切り | 「全 utils 実装 → 全テスト → 全 UI」ではなく「機能 1 つを実装 + テスト + 配線」で 1 単位 |

完了条件は必ず **実行可能なコマンド or 観察可能な挙動** で書く。「実装する」「対応する」だけのタスクは禁止。

```
NG: utils/array に分割関数を追加する
OK: utils/array/array.ts に splitMap を追加し、pnpm vitest run utils/array/array.test.ts が
    境界値（空配列・サイズ 0）込みで pass する
```

### 4. 並べる

1. 依存が満たされる順に並べる（依存先が常に先）
2. ブロッカー・高リスクが先頭
3. 各タスク完了時にリポジトリが green（`pnpm vitest run` が通る状態）を保つ
4. 3 タスク前後ごと、またはフェーズ境界にチェックポイント（`pnpm vitest run` + `pnpm run lint` + `pnpm run build` 全通し）を置く

### 5. 出力して承認を得る

出力フォーマットの Markdown を提示し、ユーザー承認を得る。永続化する場合は仕様と同じ `doc/plans/<topic>.md` に追記（または `<topic>-plan.md`）。実装は `.agents/skills/test-driven-development/SKILL.md` と `.agents/skills/incremental-implementation/SKILL.md` に従ってタスクを 1 つずつ進める。

## Hard rules

- 完了条件のないタスクを計画に入れない。完了条件は実行可能なコマンドか観察可能な挙動のみ
- 計画フェーズでコードを書かない。出力は計画ドキュメントだけ
- 依存順を無視した番号付けをしない。ブロッカー・高リスクを後回しにしない
- 1 タスクが 1 コミットに収まらない大きさなら、さらに割る（目安: 変更 5 ファイル超は分割対象）
- 検証コマンドは pnpm 表記のみ（`pnpm vitest run` / `pnpm run lint` / `pnpm run build`）
- 受け入れ基準に対応しないタスク（ついで作業・先回りの抽象化）を勝手に足さない
- 出力本文は日本語。識別子・コマンド・パスは原文のまま

## 出力フォーマット

`doc/plans/` に置ける Markdown。

````markdown
# <トピック名> 実装計画

## 目的
<仕様の要約 1–2 行。仕様: doc/plans/<topic>.md>

## タスク表

| # | 内容 | 完了条件 | 依存 |
|---|---|---|---|
| 1 | <ブロッカー/基盤タスク> | `pnpm vitest run <file>` が pass | なし |
| 2 | <タスク> | `pnpm run build` が exit 0 | 1 |
| 3 | <タスク> | <操作手順> で <観察可能な挙動> | 1 |
| C1 | チェックポイント | `pnpm vitest run` + `pnpm run lint` + `pnpm run build` 全て green | 1–3 |

## リスクと対策
- <リスク>: <先頭タスクでの検証方法 or 回避策>

## 未解決の質問
- <着手前にユーザー判断が必要な項目。なければ「なし」>
````

## 完了チェックリスト

- [ ] 仕様の受け入れ基準すべてが、いずれかのタスクの完了条件に対応している
- [ ] 全タスクに実行可能な完了条件（コマンド or 観察可能な挙動）がある
- [ ] タスクは依存順に並び、ブロッカー・高リスクが先頭にある
- [ ] 各タスクが 1 コミット相当の粒度で、単独検証可能
- [ ] フェーズ境界にチェックポイント（テスト + lint + build 全通し）がある
- [ ] ユーザーが計画を承認してから実装に入った
